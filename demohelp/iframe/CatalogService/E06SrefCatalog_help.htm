空间参照系目录服务:
一、示例功能：
该示例实现简单的空间参照系服务功能。
二、实现说明：
1、数据：使用本地数据源的GDB数据库――平台示例地理数据库sample.hdf。

2、实现步骤：

（1）先引入ol-debug.js（ol-debug是一个专为WebGIS客户端开发提供的JavaScript 类库包，用于实现标准格式发布的地图数据访问。）、jquery-1.11.2.min.js（第三方js资源库）、zdclient.js(中地公司提供的一套基于OpenLayers的MapGIS Web二次开发的脚本库)；

（2）空间参照系管理时，先初始化Zondy.Service.Catalog.GDBInfo类，然后调用类中的方法实现空间参照系服务管理。
三、功能类/参数说明
1、Zondy.Service.Catalog.GDBInfo类：

1.1 构造函数：
Zondy.Service.Catalog.GDBInfo( options)

参数说明：
options：（object类型）重新设置的该对象的属性键值对。如下所示：
【1】ip：（String类型）GIS数据服务器地址，默认是"127.0.0.1"。
【2】port：（String类型）GIS服务端口号，默认是"6163"。
【3】servername：（String类型）示例数据源名称。
【4】gdbName：（String类型）示例数据库名称。
【5】user：（String类型）用户名。，除MapGISLocal数据源，其它的都设置，默认是：""。
【6】password：（String类型）用户密码，设置数据源的用户密码，除MapGISLocal数据源，其它的都设置，默认是：""。
1.2 方法说明：
getProjectList(onSuccess)：获取参照系列表。详细参数介绍如下：
onSuccess：（Function类型）回调函数名称。

getProjectInfo(refID, onSuccess)：获取参照系信息。详细参数介绍如下：
refID：（String类型）空间参照系ID。
onSuccess：（String类型）回调函数名称。

注：详细说明请参见MapGIS IGServer JavaScript Client的API手册。
