瓦片目录服务:
一、示例功能：
该示例实现简单的瓦片服务目录管理功能。
二、实现说明：
1、数据：使用示例墨卡托瓦片数据WORLDMKT.TDF，在本地GIS服务器中发布为瓦片地图服务（WorldMKTTile）。

2、实现步骤：

（1）先引入ol-debug.js（ol-debug是一个专为WebGIS客户端开发提供的JavaScript 类库包，用于实现标准格式发布的地图数据访问。）、jquery-1.11.2.min.js（第三方js资源库）、zdclient.js(中地公司提供的一套基于OpenLayers的MapGIS Web二次开发的脚本库)；

（2）瓦片目录管理时，先初始化Zondy.Service.Catalog.TileLayerr类，然后调用类中的方法实现瓦片服务的目录管理。
三、功能类/参数说明
1、Zondy.Service.Catalog.TileLayer类：该类是一个瓦片地图目录服务类

1.1 构造函数：
Zondy.Service.Catalog.TileLayer( options)

参数说明：
options：（object类型）重新设置的该对象的属性键值对。如下所示：
【1】ip：（String类型）GIS数据服务器地址，默认是"127.0.0.1"。
【2】port：（String类型）GIS服务端口号，默认是"6163"。
1.2 方法说明：
getTileList(onSuccess)：获取瓦片服务列表。详细参数介绍如下：
onSuccess：（Function类型）回调函数名称。

setTileName(tileName)：设置目标瓦片服务名称。详细参数介绍如下：
onSuccess：（String类型）瓦片服务名称。

getTileInfo(onSuccess)：获取服务器已发布瓦片中某一个瓦片服务的详细信息。详细参数介绍如下：
onSuccess：（Function类型）回调函数名称。

注：详细说明请参见MapGIS IGServer JavaScript Client的API手册。
